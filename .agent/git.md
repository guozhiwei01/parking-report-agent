# Git 提交纪律

每完成一个完整的原子功能就提交一次。

推荐提交边界：

- 项目脚手架和依赖配置。
- 数据库模型和迁移。
- 上传 / 状态查询 / 下载 API。
- LangGraph 工作流骨架。
- 确定性分析器。
- LLM 报告规划。
- 图表和 DOCX 渲染。
- 测试。
- Docker 和 README。

规则：

- 不要堆成一个巨大的最终提交。
- 不要提交 `.env`、密钥、本地数据库文件、缓存或临时上传文件。
- commit message 要能看出功能边界。
- 不要把无关格式化或重构混进功能提交。
- 样例报告如果是交付物，可以提交到 `sample_output/`。
