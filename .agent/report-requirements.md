# 报表要求

最终输出必须是可下载的 `.docx` 报告，并且有清晰的章节结构。

六个硬指标必须从 CSV 直接计算：

1. 总交易笔数：CSV 数据行数。
2. 应收总金额：`应收金额` 求和。
3. 实收总金额：`实收金额(元)` 求和。
4. 实际抵扣总额：`实际抵扣额(元)` 求和。
5. 实收率：实收总金额 / 应收总金额 * 100，保留一位小数。
6. 主要支付方式：交易笔数最多的 `支付方式` 及其笔数。

规则：

- 不要硬编码样例数据算出来的值。
- 不要让 LLM 计算硬指标。
- 把已计算事实作为 grounded context 传给 Agent。
- DOCX 中至少包含一张真实生成的图表。
- 补充观察要简洁、面向管理者、有数据支撑。
- 结论建议要可执行，并且来自前面的分析。
