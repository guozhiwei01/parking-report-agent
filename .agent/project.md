# 项目策略

目标是做一个“小而完整、工程味足”的 Agentic 报表系统，把停车交易明细导出数据生成给管理者看的 Word 分析报告。

必须支持的生命周期：

1. 上传上下文文件：报表模板 DOCX 和交易 CSV。
2. 异步启动生成任务，提交 HTTP 请求不能被长时间阻塞。
3. 持久化并查询任务状态，刷新页面后状态不能丢。
4. 任务完成后下载生成的 DOCX 报告。

评分重点：

- 六个硬指标计算正确。
- 有真实的 Agent 工作流，并且 Agent 会做报告内容决策。
- 确定性分析和 LLM 推理边界清楚。
- 请求生命周期和 LLM 行为有结构化日志。
- 生成的 DOCX 至少包含一张真实图表。
- Docker Compose 能一键启动。
- README 简洁说明运行方式和设计思路。

不要过度实现认证、权限、复杂前端、文档预览、完整重试系统或分布式 worker，除非用户明确要求。
